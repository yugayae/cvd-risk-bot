# Руководство по миграции на модульную структуру

## ✅ Что уже сделано

1. ✅ Создана модульная структура в `frontend/js/`
2. ✅ Созданы модули:
   - `validation.js` - валидация данных
   - `api.js` - API запросы
   - `utils/normalization.js` - нормализация данных
   - `utils/interpretation.js` - интерпретации
   - `utils/form-handlers.js` - обработчики формы
   - `utils/warnings.js` - правила предупреждений
   - `render/patient-report.js` - рендеринг для пациента
   - `render/doctor-report.js` - рендеринг для врача

## 🔄 Следующие шаги

### 1. Создать constants.js с экспортом всех констант

Нужно перенести все константы из `script.js` в `frontend/js/constants.js`:

```javascript
// frontend/js/constants.js
export const MODEL_SCOPE_LABELS = { ... };
export const PATIENT_SAFETY_TEXT = { ... };
export const UI_TEXT = { ... };
// и т.д.
```

### 2. Создать main.js который использует модули

Создать новый файл `frontend/js/main.js` который:
- Импортирует все модули
- Инициализирует приложение
- Обрабатывает события формы

### 3. Обновить index.html

```html
<!-- Вместо -->
<script src="script.js"></script>

<!-- Использовать -->
<script type="module" src="js/main.js"></script>
```

## ⚠️ Важные замечания

1. **ES6 модули требуют сервер**
   - Не работают с `file://` протоколом
   - Нужен HTTP сервер (например, `python -m http.server`)

2. **Постепенная миграция**
   - Можно оставить текущий `script.js`
   - Использовать модули там, где возможно
   - Постепенно переносить функциональность

3. **Bundler (опционально)**
   - Можно использовать Webpack/Vite для сборки
   - Это позволит использовать модули везде
   - Но требует дополнительной настройки

## 🎯 Альтернативный подход

Если ES6 модули не подходят, можно:
1. Использовать IIFE (Immediately Invoked Function Expression)
2. Использовать namespace паттерн
3. Оставить текущую структуру, но улучшить организацию

Текущий код работает и так - модули это улучшение, а не требование.
